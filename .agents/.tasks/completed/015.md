# Fix bash path-validation bypasses in validateToken

## Problem

`validateToken` in the Bash command safety checks has two issues (reported by subagent review; verify before fixing):

1. Option-attached paths skip validation: any token starting with `-` is treated as "not a path" and skipped, so `--file=/etc/passwd`, `--output=/etc/x`, `-o/etc/x` embed an out-of-allowed-dir path but bypass `isPathWithinAllowedDirs`.
2. Quoted-token skip requires a newline: the code intends to skip fully quoted string arguments, but the condition is `isFullyQuoted(token) && token.includes("\n")`. A quoted string argument without a newline is NOT skipped â€” it is de-quoted and path-validated, contradicting the stated intent and potentially blocking legitimate string args.

Confidence: Medium (logic read by subagent; confirm exact line behavior during fix).

## Relevant files

- `source/utils/bash.ts` â€” `validateToken` (~L172-188)

## Fix direction

- For option-attached paths, split on `=` (and handle `-o/path` style) and validate the embedded path portion against allowed dirs.
- Reconsider the quoted-token skip: decide whether quoted args should be skipped regardless of newline, and align code with comment/intent.

## Acceptance notes

- `--file=/etc/passwd` style arguments are validated against allowed dirs.
  - `extractPathFromFlag` extracts the value after `=` (with quote stripping)
    and validates when it starts with `/`, `~`, `./`, or `../`.
- `-o/etc/passwd` style flags (single letter + attached path) are validated.
- Quoted string-argument handling matches documented intent: all fully-quoted
  tokens are skipped (removed `&& token.includes("\n")` condition).
- Added 8 new unit tests for:
  - `--file=/etc/passwd` rejection (option-attached absolute path)
  - `-o/etc/hosts` rejection (single-flag attached path)
  - `--config=./relative/path` acceptance (relative path within project)
  - `--config=~/home-path` rejection (home dir outside project)
  - `--label=name/value` skipping (non-path string in flag value)
  - unquoted absolute path still validated
  - fully-quoted token treated as string argument (not validated)
  - fully-quoted multiline string still skipped
- All 113 bash util tests pass.
- `npm run check` passes (typecheck, lint, format).
