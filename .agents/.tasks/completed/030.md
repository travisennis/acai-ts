# Improve markdown table rendering with content-aware column widths

## Problem

Markdown tables rendered to the terminal were hard to read and wasted space:

- Column widths were not dynamic to content. `Markdown.renderTable` forced every
  column to the same width (`floor((80 - 5) / columnCount)`), so short columns
  (e.g. `Status`) wasted space while wide columns (e.g. `Description`) over-wrapped.
- Table width did not use the terminal. The total was derived from a hardcoded
  `80`, ignoring the real terminal width, so wide content never used a 120+ column
  terminal.
- Text wrapping/truncation was poor as a direct result of the equal-width split.

## Relevant files

- `source/tui/components/markdown.ts` — `renderTable`, `renderBlockTableToken`,
  and the `table` case in `renderToken`.
- `test/tui/markdown-render-token.test.ts` — table-rendering tests.

## Fix direction

- Thread the real available width into `renderTable` (it previously received none,
  hence the hardcoded `80`).
- Measure each column's natural width (longest line) and minimum width (longest
  word) using `visibleWidth` so ANSI styling and wide characters are counted.
- Keep natural widths when the table fits; otherwise shrink the widest columns
  first (with word wrapping) so content-heavy columns share space fairly while
  narrow columns stay compact.
- Truncate unbreakable tokens with an ellipsis so the table never exceeds the
  available width (and the renderer's post-wrap pass leaves it intact).
- Honor GFM column alignment (`:---`, `---:`, `:--:`).

## Acceptance notes

- Columns are sized to content: narrow columns stay narrow, the content-heavy
  column gets the remaining room (verified at width 120).
- Table width tracks the terminal: wide content grows past the old ~80 cap and up
  to the full terminal width; compact tables stay compact rather than stretching.
- No rendered line exceeds the requested width on narrow terminals; tight columns
  wrap on word boundaries, and unbreakable tokens truncate with `…`.
- GFM column alignment is applied (numeric columns right-align).
- Added 5 tests to `test/tui/markdown-render-token.test.ts` covering proportional
  sizing, no-overflow on narrow terminals, growth beyond the old cap, staying
  within terminal width, and alignment.
- Full suite green: 1158 tests pass; `npm run typecheck` and biome lint/format
  clean.
- Delivered in commit "feat(markdown): size table columns to content and
  terminal width" (this task record is included in the same commit).

Note: No ADR was created. The change is an internal rendering-algorithm
improvement to an existing feature; the public interfaces (`Markdown` component
and `Table` API) are unchanged, so no durable architectural contract is
introduced. Flagging in case an ADR is desired for the terminal-rendering
behavior change.
