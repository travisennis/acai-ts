# Fix rm -rf protection bypasses in command-protection

## Problem

`detectDangerousRmRf` has three bypasses, all verified by reading the logic:

1. Multi-path: `afterRmRf` captures the entire argument string and `isTempDirectoryOnly` only checks `afterRmRf.startsWith("/tmp/")`. So `rm -rf /tmp/cache /home/user/important` passes the temp-only check and is NOT blocked, deleting the non-temp path.
2. Path traversal: the check is a pure string prefix test with no normalization, so `rm -rf /tmp/../etc` and `rm -rf /tmp/../../home` are treated as temp-only and not blocked even though they resolve outside `/tmp`.
3. Flag variants: the detector keys off `/rm\s+-rf\s+/i`, so equivalent destructive forms slip through: `rm -fr /`, `rm -rfv /home`, `rm -r -f /`, `rm --recursive --force /`.

Confidence: High (security-sensitive guard bypass).

## Relevant files

- `source/utils/command-protection.ts` — `detectDangerousRmRf` (~L256-309), match regex at ~L258

## Fix direction

- Tokenize the arguments and validate EACH path is temp-only, not just the joined string.
- Normalize/resolve paths before the temp-dir prefix check to defeat `..` traversal.
- Match recursive+force flags robustly (any order, combined or separate, long forms): detect `rm` with recursive (`-r`/`-R`/`--recursive`) AND force (`-f`/`--force`) including combined clusters like `-rf`/`-fr`/`-rfv`.

## Acceptance notes

- Implemented `rm` command scanning that detects recursive+force flags in combined, separated, reordered, verbose, and long-option forms.
- Validates every rm operand instead of only the joined argument string, so `rm -rf /tmp/x /home/y` is blocked.
- Normalizes absolute path operands before temp-dir checks, so `rm -rf /tmp/../etc` is blocked.
- Preserves allowed temp-only removals for `/tmp`, `/var/tmp`, `$TMPDIR`, and `${TMPDIR}` paths, including multi-path temp-only commands.
- Added unit tests covering `rm -rf /tmp/x /home/y`, `rm -rf /tmp/../etc`, `rm -fr /`, `rm -rfv /home`, `rm -r -f /`, `rm --recursive --force /`, and a multi-temp-path allowed command.
- Reused the hardened rm detector for inline script and heredoc script-content scanning, with tests for `bash -c "rm -fr /"` and heredoc `rm --recursive --force /`.
