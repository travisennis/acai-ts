# Log per-turn input token time series

- Status: Pending
- Priority: P2
- Effort: S
- ExecPlan: none
- Depends on: none

## Problem

Sessions today only record `tokenUsage.lastTurn.inputTokens`. We can see the final turn's size but not how context grew over the session. Plotting context growth turn-by-turn would let us quantify "context inflation" (`analysis.md` §7) and correlate it with wall-time slowdowns later in a session.

## Relevant files

- `source/agent/` — turn loop where token usage is observed
- `source/session/` — session record persistence
- `~/.acai/sessions/session-*.json` — session schema

## Implementation notes

For each assistant turn append a record to the session with at minimum:

- `turnIndex`
- `inputTokens` (cached + uncached split if the provider returns it)
- `outputTokens`
- `reasoningTokens`
- `cachedInputTokens` if available
- `wallMs` for the turn (start of model request → end of tool result handling)

A simple array of per-turn entries on the session record is enough. No UI changes required.
