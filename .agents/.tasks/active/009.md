# Fix apply_patch path truncation on paths containing a colon

## Problem

`parsePatchHeader` extracts file paths with `line.split(":", 2)[1]`. JS `split` with a limit discards everything after the second colon, so a path containing a colon is truncated. Verified: `"*** Add File: C:/x".split(":", 2)` returns `["*** Add File", " C"]`, so the path becomes `"C"`.

This corrupts any Windows-style absolute path or any path containing `:` for `*** Add File:`, `*** Delete File:`, `*** Update File:`, and `*** Move to:`.

Confidence: High (verified by execution).

## Relevant files

- `source/tools/apply-patch.ts` — `parsePatchHeader`, lines ~71, 77, 81, 86

## Fix direction

Capture the remainder after the first colon, e.g. `line.slice(line.indexOf(":") + 1).trim()`, for each header variant.

## Acceptance notes

- A patch targeting a path containing a colon resolves the full path.
- Add a unit test covering a colon-containing path for add/update/delete/move headers.
- `npm run check` passes.
