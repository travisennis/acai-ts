# Surface Bash exit code and duration prominently in tool result

- Status: Pending
- Priority: P2
- Effort: S
- ExecPlan: none
- Depends on: none

## Problem

In the analyzed session two parallel `rg` commands hit the 90s default timeout (`duration: 90005`, exit 1) and the agent then re-issued essentially the same search with `grep -rn`. The model did not appear to recognize "this exited 1 after 90s — change strategy". Surfacing exit code and duration prominently in the tool result text (not just in metadata) would let the model adapt faster and would also give us a clean signal in logs/sessions when investigating timeouts.

See `analysis.md` §3.

## Relevant files

- `source/tools/bash.ts` (or the equivalent tool definition)
- Any tool that runs subprocesses (test runners, formatters)

## Implementation notes

When a Bash result is returned to the model, prefix or suffix the result body with a short, parseable line such as:

```
[exit=1, duration=90005ms, timed_out=true]
```

Include `timed_out=true` when the duration matches the configured timeout. Keep the format stable so it can be both read by the model and grepped from logs.

This is intentionally a presentation-only change. Do not change timeout defaults here — that is a separate remediation task tracked in the broader analysis follow-ups.

## Acceptance notes

- Bash tool results visible to the model include a one-line exit/duration/timeout summary.
- A unit test covers the success, failure, and timeout cases.
- `npm run check` passes.
