# Fix broken markdown link detection regex

## Problem

`markdownLinkRegex = /\((.*?)\]\((.*?)\)/` starts with `\(` (a literal opening paren) where it should be `\[`. A standard markdown link `[text](url)` does not match. Verified: current pattern returns `false` on `[a](b)`; the corrected `/\[(.*?)\]\((.*?)\)/` returns `true`.

Markdown link detection in `isMarkdown` is therefore broken; links only register when other heuristics happen to fire.

Confidence: High (verified by execution).

## Relevant files

- `source/terminal/markdown-utils.ts` — L5

## Fix direction

Change the leading `\(` to `\[`: `/\[(.*?)\]\((.*?)\)/`.

## Acceptance notes

- `isMarkdown("[text](https://example.com)")` returns true.
- Add a unit test for link detection.
- `npm run check` passes.
