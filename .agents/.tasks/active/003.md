# Log tool-call serialized size

- Status: Pending
- Priority: P2
- Effort: S
- ExecPlan: none
- Depends on: 001

## Problem

Edit-decode time is suspected to be the largest single contributor to wall time (~12 of 34 minutes in the analyzed session, `analysis.md` §2). The biggest single Edit was 7,572 chars of model-generated `oldText` + `newText`. We need to actually measure this per call so we can confirm or refute the theory and identify pathological tools.

## Relevant files

- `source/agent/` — tool dispatch / response handling
- `source/tools/` — individual tool definitions (Edit, Bash, Read, etc.)
- `source/utils/logger.ts` — structured event emission

## Implementation notes

For each tool call emit a structured log event including:

- `tool` name
- `tool_call.input_size` (serialized JSON byte/char length of the model-generated arguments)
- `tool_call.output_size` (size of the tool result returned to the model)
- `tool_call.duration_ms`
- correlation id linking to the model request that produced it (so it joins cleanly with the timing telemetry from task 001)

Aggregate-friendly format: one JSON line per call.

## Acceptance notes

- A real session produces a `tool_call` log event per tool invocation with the fields above.
- Documented alongside the model timing telemetry from task 001.
- `npm run check` passes.
