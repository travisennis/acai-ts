# Add session persistence round-trip tests

## Problem

Session persistence (`source/sessions/manager.ts`) has basic unit tests for append/get/clear operations but zero tests for the critical persistence round-trip: `save()` → `load()` → `restore()` produces identical state. The migration from old `TokenUsageTurn[]` to new `SessionTokenUsage` format is also untested, and session resumption is the most user-visible persistence path.

## Relevant files

- `source/sessions/manager.ts` — SessionManager
- `test/sessions/manager.test.ts` — existing tests

## Implementation notes

1. Add round-trip tests covering:
   - Empty session (no messages) → save → load → restore
   - Session with user + assistant messages → save → load → restore
   - Session with tool calls and results → save → load → restore
   - Session with image parts → save → load → restore
   - Old-format token usage (TokenUsageTurn array) → load → restore → new format
   - Session with all message types
2. Use a temp directory via `fs.mkdtempSync()` for the stateDir, and clean up after each test.
3. After `restore()`, assert:
   - `get().length` matches original
   - `getSessionId()` is preserved
   - `getTitle()` is preserved
   - `getTotalTokenUsage()` matches
4. Verify that `save()` does not throw and the JSON file exists at the expected path.

## Acceptance notes

- At least 5 round-trip test cases pass.
- Old-format token usage migrates correctly.
- `npm run check` passes.
