# Fix transient messages potential duplicate in SessionManager.get()

## Problem

In `source/sessions/manager.ts:179-184`, the `get()` method inserts transient messages before the last history element without checking whether the transient messages and the last element have the same role. If `transientMessages` contains a user message and the last history entry is also a user message, the model sees two consecutive user messages. While the AI SDK merges consecutive same-role messages in most cases, this is inconsistent and could cause issues with some providers or when debugging session output.

## Relevant files

- `source/sessions/manager.ts` — `get()` method

## Implementation notes

1. Before inserting transient messages, check the role of the last history element.
2. If both are `user` role, merge the transient message content into the last history element instead of inserting separately:
   ```typescript
   // If transientMessages has user messages and last history entry is also user,
   // merge the transient content into the last history entry
   if (
     this.transientMessages.length > 0 &&
     history.length > 0 &&
     this.transientMessages[this.transientMessages.length - 1]?.role === "user" &&
     history[lastIndex]?.role === "user"
   ) {
     // merge transient content into last entry
     // ...handle merging TextParts
   }
   ```
3. Keep the existing behavior (insert before last) for all other role combinations.
