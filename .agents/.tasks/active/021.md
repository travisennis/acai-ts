# Add characterization tests for the agent loop

## Problem

The agent loop (`source/agent/index.ts`, 1108 lines) is the orchestrator for the entire application — tool execution, error recovery, abort handling, streaming chunks, and iteration control. Its test file (`test/agent/index.test.ts`) contains only trivial constructor tests. Every bug fix or feature addition to the loop is high-risk without a regression safety net.

## Relevant files

- `source/agent/index.ts` — the agent loop
- `test/agent/index.test.ts` — existing (trivial) tests
- `source/tools/types.ts` — tool types
- `source/sessions/manager.ts` — session manager (dep of agent)

## Implementation notes

1. Extract testable sub-units:
   - `validateToolInputs` (pure function, easy to test)
   - `formatToolResult` (pure function)
   - `handleStreamChunk` (state machine with clear input/output)
2. Then write characterization tests for the `run` generator:
   - Single user message → assistant response (no tool calls)
   - Single tool call → tool execution → result
   - Multiple parallel tool calls
   - Abort signal during streaming
   - Error recovery (InvalidToolInputError)
   - Error exhaustion (exceed maxRetries)
3. Use mocked `ModelManager`, `SessionManager`, and `TokenTracker`. The `toAiSdkTools` utility already exists for test tool construction.
4. Follow the existing test patterns in `test/commands/session-command.test.ts` for mocking structure.

## Acceptance notes

- At least 6 test cases covering the paths above.
- All tests pass with `node --test`.
- The existing test file structure is preserved (add new tests alongside existing ones).
- `npm run check` passes.
