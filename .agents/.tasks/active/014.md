# Fix unordered-list markdown regex over-matching inline text

## Problem

`markdownUnorderedListRegex = /^-\s|\*\s|\+\s/m` parses as `(^-\s) | (\*\s) | (\+\s)`. Only the `-` alternative is anchored to line start; `* ` and `+ ` match anywhere on a line. Verified: inline text like `price is 3 * 4` is misdetected as markdown (returns `true`); the intended `/^(?:-|\*|\+)\s/m` returns `false`.

Confidence: Medium (verified by execution; causes false-positive markdown detection).

## Relevant files

- `source/terminal/markdown-utils.ts` — L7

## Fix direction

Anchor all alternatives to line start: `/^(?:-|\*|\+)\s/m`.

## Acceptance notes

- `isMarkdown("price is 3 * 4")` is not driven true by the list heuristic.
- Real list lines (`- item`, `* item`, `+ item`) still detected.
- Add a unit test covering both cases.
