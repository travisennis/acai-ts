# Add audit log redaction for sensitive content

## Problem

The audit message middleware (`source/middleware/audit-message.ts`) writes full prompt contents (including file contents loaded via `#file` mentions) to disk for every model request. If a user's prompt processes a file containing secrets (API keys, credentials, etc.), those secrets end up in the audit log on disk. The audit log path is configurable, but there is no way to redact sensitive content.

## Relevant files

- `source/middleware/audit-message.ts` — the audit middleware

## Implementation notes

1. Add an optional `redactPatterns` configuration option to the audit middleware factory.
2. When enabled, apply the patterns (RegExp[] or string[]) against the prompt text before writing to disk.
3. Common patterns to include by default: `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `EXA_API_KEY`, `JINA_READER_API_KEY` (env var value patterns).
4. The redaction should replace matched values with `[REDACTED]`.
5. Default behavior (no patterns) should remain as-is to avoid breaking existing users who rely on full audit logs.

## Acceptance notes

- When redact patterns are configured, matching content in audit log files is replaced with `[REDACTED]`.
- Default (no patterns) behavior is unchanged.
- `npm run check` passes.
