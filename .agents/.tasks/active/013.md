# Fix template placeholder $1 corrupting $10 and higher

# Fix template placeholder $1 corrupting $10 and higher

## Problem

`replaceArgumentPlaceholders` loops `i` from 0 upward and does `result.replaceAll("$" + (i+1), args[i])`. Because `replaceAll` is plain substring matching and `$1` is processed before `$10`, the `$1` inside `$10` is rewritten first. Verified: with 10 args, `$10` becomes `<arg0>0` instead of arg index 9. Any template using `$10`+ positional args is corrupted.

Confidence: Medium (clear logic flaw; severity depends on whether commands use ≥10 positional args).

## Relevant files

- `source/utils/templates.ts` — `replaceArgumentPlaceholders`, L10-16

## Fix direction

Replace placeholders in descending index order (highest first), or use a single regex pass with a replacer function that maps `$N` to the correct argument.

## Acceptance notes

- A template referencing `$1`..`$12` with 12 args substitutes each correctly.
- Add a unit test with ≥10 arguments.
- `npm run check` passes.
