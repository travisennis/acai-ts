# Address 3 high-severity transitive npm advisories

## Problem

As of 2026-06-20, `npm audit --omit=dev` reports 3 HIGH-severity vulnerable packages in transitive dependencies:

- **@hono/node-server** (`<=1.19.12`): authorization/static-path bypasses in `serveStatic`.
- **hono** (`<=4.12.24`): multiple advisories including JWT Authorization scheme handling, cookie/header handling, static-file traversal, cache leakage, body-limit bypasses, and JSX injection issues.
- **undici** (`7.0.0 - 7.27.2`): WebSocket parser/DoS issues, HTTP request/response smuggling, CRLF/header injection, cache disclosure, response queue poisoning, and SameSite downgrade issues.

Both are pulled in transitively via `@ai-sdk/devtools` and the core AI SDK.

## Relevant files

- `package.json` — direct dependencies
- `package-lock.json` — transitives

## Implementation notes

1. Run `npm audit --omit=dev` and record the full advisory list.
2. Check if updating `@ai-sdk/devtools`, `@ai-sdk/openai`, `@ai-sdk/anthropic`, etc. to their latest versions resolves the transitives.
3. If not, add an `overrides` entry in `package.json` for the affected transient deps using currently patched versions reported by `npm audit fix --dry-run` or the advisory pages. Do not pin to the older example ranges from the original audit; they are now stale.
4. Run `npm install` and verify `npm audit` is clean (or reduced).
5. Run full test suite to confirm no regressions from dep bumps.
