# Fix agent consecutiveErrors never resetting on success

# Fix agent consecutiveErrors never resetting on success

## Problem

In the agent loop `consecutiveErrors` is declared at ~L204, incremented only in the catch block (~L349), and the loop aborts when `consecutiveErrors > maxRetries` (~L420). It is never reset to 0 after a successful iteration. The success path does `iter += 1` (~L347) and loops again without touching `consecutiveErrors`.

As a result it behaves as a cumulative error budget, not a consecutive-retry counter, despite the variable name and the log strings ("attempt N/maxRetries+1"). A long run with scattered, recovered errors will terminate early with "Exceeded maximum retry attempts."

Confidence: High (verified there is no reset anywhere in the function).

## Relevant files

- `source/agent/index.ts` — declaration ~L204, increment ~L349, threshold ~L420, success path ~L347

## Fix direction

Reset `consecutiveErrors = 0` on the success path (e.g. immediately before/after `iter += 1`).

## Acceptance notes

- Errors that are separated by successful iterations no longer accumulate toward the retry limit.
- Add/extend a test simulating error -> success -> error and asserting the loop is not aborted prematurely.
- `npm run check` passes.
