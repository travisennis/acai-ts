# Fix unbounded step accumulation in agent state

## Problem

In `source/agent/index.ts:186-187`, the agent loop pushes `{ toolCalls, toolResults }` entries into `this._state.steps` every iteration but never clears them. Each item is a small object, so the leak is not catastrophic, but over long sessions (hundreds of iterations) unnecessary memory accumulates and the `state` object is serialized/sent to the TUI for display.

## Relevant files

- `source/agent/index.ts` — `run()` method and `resetState()`

## Implementation notes

1. In `resetState()`, already clears `this._state.steps = []`. The issue is not reset but accumulation within a single `run()` call.
2. The purpose of `steps` is to track tool calls per step for the active `run()` invocation. Since `run()` is called fresh each time the user submits input, `steps` starts fresh each call — the leak is within a single `run()` of many iterations.
3. Fix: after the while loop finishes (before `agent-stop` is emitted), clear `this._state.steps` or limit it to the last N steps (e.g., last 5). Or only expose the step data that the TUI needs rather than the full array.
4. Simplest fix: `this._state.steps = []` after the while loop exits, before yielding `agent-stop`.

## Acceptance notes

- `this._state.steps` does not grow unboundedly within a single agent run.
- Existing tests pass.
- TUI display of tool execution is unaffected (it renders from live events, not step history).
- `npm run check` passes.
